# Hongwei Portfolio Website - Project Brief

## 1. Project Overview

This project is a personal portfolio website for Hongwei Zhao.

The purpose of the website is to present Hongwei's technical background, projects, skills, certifications, education, and work experience to employers, recruiters, and professional contacts.

The website should look professional, clean, and modern. It should emphasize Hongwei's strengths in cloud computing, DevOps, IT support, full-stack development, and project-based learning.

## 16. Development Rules

When continuing this project with ChatGPT or Codex:

- Make small changes one step at a time.
- Review code before accepting changes.
- Keep components reusable when possible.
- Keep project data in separate files when the page becomes too large.
- Avoid adding unnecessary packages.
- Run the development server after major changes.
- Run a production build before deployment.
- Commit changes regularly.
- Keep sensitive values out of GitHub.
- Do not commit `.env.local`.
- Do not commit API keys, Firebase private keys, AWS credentials, kubeconfig files, or database URLs.
- Use public GitHub repositories only when they contain no secrets.

- Keep the avatar in the hero section, not in the navbar.
